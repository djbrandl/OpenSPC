# UI Design Specification: Characteristic Configuration Redesign

**Version**: 1.0
**Date**: 2026-02-05
**Author**: UI Designer
**Status**: Pending CEO Approval

---

## Executive Summary

Redesign the CharacteristicForm component from a long vertical scroll to a **tabbed interface** with **progressive disclosure via accordions** within each tab. This improves navigation, reduces cognitive load, and groups related settings logically.

---

## Design Direction

**Aesthetic**: Industrial/Utilitarian with refined details
**Tone**: Professional, data-focused, enterprise-grade
**Key Principle**: Progressive disclosure - show essentials first, reveal complexity on demand

## Tab Structure

### Tab 1: General
Primary identification and display settings.

| Section | Default State | Contents |
|---------|---------------|----------|
| Identity | **Expanded** | Name, Description, Provider (badge), Hierarchy (breadcrumb) |
| Display Options | Collapsed | Decimal precision, unit of measure (future) |
| Advanced Metadata | Collapsed | Created date, last modified, sample count (read-only) |

### Tab 2: Limits
Specification and control limits management.

| Section | Default State | Contents |
|---------|---------------|----------|
| Specification Limits | **Expanded** | Target, USL, LSL with visual indicator |
| Control Limits | **Expanded** | UCL, LCL (read-only) + Recalculate button |
| Limit Visualization | Collapsed | Mini preview chart showing limit positions |

### Tab 3: Sampling
Subgroup configuration and scheduling.

| Section | Default State | Contents |
|---------|---------------|----------|
| Subgroup Settings | **Expanded** | Mode selector, min measurements, warn threshold |
| Stored Parameters | Conditional | Only for STANDARDIZED/VARIABLE modes |
| Schedule (MANUAL only) | **Expanded** | Full schedule configuration component |

### Tab 4: Rules
Nelson rules and violation detection settings.

| Section | Default State | Contents |
|---------|---------------|----------|
| Nelson Rules | **Expanded** | All 8 rules in a compact grid/card layout |
| Quick Actions | Static | "Enable All" / "Disable All" / "Reset to Defaults" |

      <TabPanel id="limits">
        <LimitsTab formData={formData} characteristic={characteristic} />
      </TabPanel>
      <TabPanel id="sampling">
        <SamplingTab
          formData={formData}
          scheduleConfig={scheduleConfig}
          isManual={characteristic.provider_type === 'MANUAL'}

  <Footer>

## Interaction Behaviors

### Tab Navigation
- Tabs are keyboard accessible (arrow keys)
- Active tab persists in URL query param (`?tab=limits`)
- Unsaved changes prompt before switching tabs

### Accordion Behavior
- Multiple accordions can be open simultaneously
- Open state persists during session
- Smooth expand/collapse animation (200ms)

### Form State
- Dirty indicator shows in footer when changes exist
- Cancel confirms if dirty
- Save validates all tabs before submission
- Tab with validation errors highlighted

### Responsive Behavior
```
Desktop (>1024px): Side-by-side layout within accordions
Tablet (768-1024px): Stacked layout, tabs remain horizontal
Mobile (<768px): Tabs become scrollable horizontal strip
```
